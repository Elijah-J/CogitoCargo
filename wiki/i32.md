# `i32`

`i32` is Rust's signed 32-bit integer type. It is the first integer type
written explicitly in `hello_type_annotation`.

## Shape I have used

```rust
let count: i32 = 3;
println!("Count: {count}");
```

`count` is the binding. `: i32` is the type annotation. `3` is the integer
literal value.

## Default integer inference

For an unsuffixed integer literal with no stronger type information, Rust
defaults to `i32`:

```rust
let count = 3;
```

The Rust Reference describes this as the fallback when program context
under-constrains the integer type. If context uniquely determines another
integer type, the literal has that type instead.

```rust
let count: i32 = 3;
```

This version does not rely on the default. The `: i32` annotation writes the
type directly in source code.

## Function signature shape

```rust
fn add_one(number: i32) -> i32 {
    number + 1
}
```

The first `i32` is the parameter type for `number`. The second `i32` is the
return type after `->`.

## Useful guardrail

`i32` is a type name, not a value. In `let count: i32 = 3;`, `i32` describes
the kind of integer value, and `3` is the value.
